# Vibe コーディング ラボの演習のスコープを設定する方法

Vibe コーディング ラボの演習は、参加者が限られた時間内に概念を効果的に学習して適用できるよう、特定のタスクと結果を対象とするようスコープを設定する必要があります。 

## Vibe コーディング プロジェクトの種類

ラボ演習のコーディング プロジェクトは、20 から 30 分以上の演習が収まるよう、よく考えてスコープを厳密または具体的に設定する必要があります。

1. プロジェクトを最初から始める (基礎、スタブ機能、ページ ナビゲーションを実装しますが、アプリ データはほとんどまたはまったく実装しません)

    1. プロトタイプの eコマース アプリを作成する

        プロンプトの例:

        - JavaScript、HTML、CSS を使ってプロトタイプの eコマース アプリを作成する必要があります。
        - プロトタイプの eコマース アプリは、デスクトップと電話デバイスでの表示に対応するため、自動的に拡大縮小する動的なユーザー インターフェイスを備えている必要があります。
        - eコマース アプリには、商品リスト、商品詳細、ショッピング カート、チェックアウトのページが含まれている必要があります。 各ページは、基本的な機能と、ページ間の前後のナビゲーションを備えている必要があります。
        - 10 個の果物商品の簡単なデータセットを使います。 商品名、説明、単位あたりの価格 (単位は品目数やオンス、ポンドなど)、商品を表す簡単な画像を含みます。
        - 商品一覧ページには、商品の一覧と、商品名、単価、画像などの基本情報が表示される必要があります。 商品一覧ページには、商品の数量を選ぶ方法と、選んだ品目をショッピング カートに追加するオプションも用意されている必要があります。
        - 商品詳細ページには、商品名、説明、単価、画像など、選んだ商品に関する詳細情報が表示される必要があります。 商品詳細ページには、商品一覧ページに戻る方法も用意されている必要があります。
        - ショッピング カート ページには、商品名、数量、合計価格など、カートに追加された商品の一覧が表示される必要があります。 ショッピング カート ページには、カート内の商品の数量を更新する方法と、カートから商品を削除する方法も用意されている必要があります。
        - チェックアウト ページには、商品名、数量、合計価格など、カート内の商品の概要が表示される必要があります。 チェックアウト ページには、注文を確定する方法も用意されている必要があります。 
        - プロトタイプの eコマース アプリは、ページ間の基本的なナビゲーションを備えている必要があります。 商品一覧の左側の折りたたみ可能なナビゲーション バー、商品詳細、ショッピング カート、チェックアウト ページを作成します。 ユーザーは、ナビゲーション バーを使って、商品一覧、商品詳細、ショッピング カート、チェックアウトのページ間を移動できる必要があります。
        - プロトタイプの eコマース アプリは、魅力的な見た目にするための基本的なスタイルを備えている必要がありますが、完全な応答性や洗練さは必要はありません。
        - プロトタイプの eコマース アプリには、ユーザー認証、支払い処理、データベース統合などのバックエンド機能は含まれません。 これは、eコマース アプリの基本的な概念を見てもらうための静的なプロトタイプです。

    1. AI で機能を強化されたアプリのプロトタイプを作成する (Python と Hugging Face モデルを使用)

1. 新しい機能をプロジェクトに追加する

    1. 既存のアプリに新しい機能を追加する
        - たとえば、検索機能を eコマース アプリに追加します。
        - ユーザーは検索機能を使って名前または説明で商品を検索できる必要があります。
        - 検索結果では、検索条件に一致する商品の、商品名、単価、画像を含む一覧が表示される必要があります。
        - 検索機能は、商品一覧ページからアクセスでき、商品一覧ページに戻る方法を備えている必要があります。

    1. 既存のアプリに新しいページを追加する

        - たとえば、問い合わせページを eコマース アプリに追加します。
        - 問い合わせページには、ユーザーが自分の名前、メール アドレス、メッセージを送信できるフォームが含まれている必要があります。
        - 問い合わせページには、商品一覧ページに戻る方法も用意されている必要があります。

1. ツール、フレームワーク、またはテクノロジ プロジェクトを変更する

    1. 既存のアプリのフロントエンド フレームワークを変更する

        - たとえば、eコマース アプリのフロントエンド フレームワークを JavaScript から React または Angular に変更します。
        - 新しいフロントエンド フレームワークでは、商品一覧、商品詳細、ショッピング カート、チェックアウトなど、元のアプリと同じ機能を提供する必要があります。
        - 新しいフロントエンド フレームワークは、デスクトップと電話デバイスでの表示に対応するため、自動的に拡大縮小する動的なユーザー インターフェイスを備えている必要もあります。

    1. 既存のアプリのバックエンド テクノロジを変更する

        - たとえば、eコマース アプリのバックエンド テクノロジを Node.js から Python Flask または Django に変更します。
        - 新しいバックエンド テクノロジでは、ユーザー認証、支払い処理、データベース統合など、元のアプリと同じ機能を提供する必要があります。

1. コード レビューと改善プロジェクト

    1. プロジェクトにログを追加する
    1. プロジェクトに単体テストを追加する

1. コーディング言語変換プロジェクト

    既存のアプリを別の言語またはフレームワークに変換します。

    たとえば、Python アプリを Java アプリまたは C# アプリに変換します。

1. アーキテクチャ再設計プロジェクト (大きすぎる)
